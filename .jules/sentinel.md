## 2025-02-24 - Enforce HTTPS by Disabling Cleartext Traffic
**Vulnerability:** The Android application allowed cleartext traffic by setting `android:usesCleartextTraffic="true"` in the `AndroidManifest.xml`. This could allow attackers on the same network to intercept or modify unencrypted network communications, leading to potential Man-in-the-Middle (MITM) attacks and data exposure.
**Learning:** Some applications might enable cleartext traffic for debugging or older non-HTTPS API dependencies, but keeping it true in production exposes the app to significant security risks, especially if sensitive data is transmitted.
**Prevention:** Always enforce HTTPS by explicitly setting `android:usesCleartextTraffic="false"` in the `AndroidManifest.xml` (the default on Android 9+). Make sure all external API and web traffic requests use HTTPS. For local debugging, specific exceptions can be made via a Network Security Configuration file rather than globally enabling cleartext traffic.
